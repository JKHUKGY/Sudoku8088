# 理解招聘流程：你的简历到底给谁看？

> 这是「用 AI 构建你的简历」系列课程的第一节。在我们开始使用工具之前，先从基础讲起：理解美国公司的招聘到底是怎么运作的。

## 课程导读

欢迎来到这个系列课程的第一节。

我们的学习路径是这样的：前几节课专注于理论和知识——帮你理解什么是好的简历，以及为什么。之后，我们再进入实操部分：用 AI 工具来管理、迭代、持续优化你的简历。

为什么要从招聘流程讲起？因为简历不只是一份文档，它是写给特定读者的一封信。你不知道读者是谁，就写不出打动人的内容。

---

## 学习目标

想一想：你花了好几个小时打磨简历，但你知道到底是谁在看吗？

是一个 Recruiter，每天扫几百份简历？是一个自动化系统，按关键词过滤？还是一个 Hiring Manager，想找人解决具体问题？

答案不同，你的简历写法就完全不同。

学完这节课，你会：

1. 理解三种不同的招聘场景，以及简历在每种场景中的流转路径
2. 认识招聘流程中的关键角色：谁在筛选、谁在做决定
3. 意识到一个重要现实：应届生投中小公司时，是在和有经验的人同池竞争

---

## 核心概念

### 核心问题：你的简历给谁看？

在你写下任何一条经历之前，先问自己：

- 这份简历会经过谁的手？
- 谁在筛选？谁在做决定？
- 他们在找什么？

不同的招聘场景，意味着不同的读者。理解这一点，你的整个写简历思路都会变。

---

### 场景一：大公司校园招聘

**典型公司**：FAANG、Big 4 会计师事务所、大型银行、知名科技公司

**谁在招人**：University Recruiting 团队——专门负责校招的 HR 部门

**流程是怎样的**：

1. 公司确定明年的 headcount（招聘名额）
2. University Recruiting 团队选择 target school（目标学校，通常分等级）
3. 去学校参加 Career Fair、办 Info Session
4. 通过学校平台（如 Handshake）或公司官网收简历
5. 大量简历进入系统，进行标准化筛选（ATS 系统 + 人工）
6. 通过的人进入 Online Assessment → Phone Screen → On-site 面试
7. 批量发 Offer，统一入职时间

**这种模式的特点**：

- 批量招聘，流水线作业，流程标准化
- 硬指标权重很高：学校名气、GPA、专业、关键词匹配
- 没有具体的 Hiring Manager 从头盯着你——你是先被筛选，录用后才分配到具体团队

**对简历意味着什么**：你的第一关是通过标准化筛选。简历要「看起来合格」，让 ATS 和 Recruiter 能快速打勾。

---

### 场景二：中小公司岗位招聘（应届生直接投递）

**典型公司**：初创公司、中型科技公司、没有专门校招流程的公司

**谁在招人**：Hiring Manager（用人经理）有需求，自己看简历或者让 Recruiter 帮忙筛

**流程是怎样的**：

1. Hiring Manager 缺人了（团队人手不够、有新项目要启动）
2. 写 Job Description，发到 LinkedIn、公司官网、Indeed
3. 简历陆续进来，可能几十份，可能几百份
4. Recruiter 或 Hiring Manager 亲自筛选
5. 电话面试 → 技术面试 → On-site → Offer

**这种模式的特点**：

- **应届生和有经验的人在同一个池子里竞争**
- 没有「校招专属通道」，你的简历和工作了 2-3 年的人放在一起比
- 流程更灵活，Hiring Manager 的影响力更大

**对简历意味着什么**：你要能脱颖而出。没有工作经验，就要靠项目、潜力、学习能力让 Hiring Manager 觉得：「这个人值得聊一聊。」

**重要提醒**：很多应届生在北美求职时，投的其实是这种岗位。别以为只有大公司校招才能找到工作。但这也意味着——你的简历要更有竞争力。

---

### 场景三：社招（有经验者跳槽）

**典型情况**：有 1-5 年工作经验的人换工作

**谁在招人**：Hiring Manager 有具体需求，找 Internal Recruiter 或 Agency Recruiter 帮忙

**流程是怎样的**：

1. Hiring Manager 明确需要什么样的人（技能、经验、级别）
2. Recruiter 发布职位、主动 sourcing（找候选人）
3. 简历大量涌入（热门岗位可能 1000+）
4. Recruiter 筛到 20-30 份，交给 Hiring Manager
5. Hiring Manager 挑人进面试流程
6. 面试 → Offer

**这种模式的特点**：

- Hiring Manager 有具体问题要解决（缺某个技能、补某个位置）
- 你的简历要展示：你能解决他的问题
- 相关经验和具体成果非常重要

**对简历意味着什么**：展示你有相关经验、能上手解决问题。不是「我很厉害」，而是「我能帮你解决 X 问题」。

---

## 关键角色

### Hiring Manager（用人经理）

- 真正需要招人的那个人
- 最终拍板决定要不要你
- 他关心的是：这个人能不能解决我的问题？能不能融入团队？

### Recruiter（招聘人员）

- 帮 Hiring Manager 筛简历的人
- 是 gatekeeper（守门人），你的简历要先过他这关
- 分两种：
  - **Internal Recruiter**：公司内部 HR，了解公司文化和需求
  - **Agency Recruiter**：外部猎头，按成功入职收费

### ATS（Applicant Tracking System）

- 自动化简历管理系统
- 帮助筛选、排序、追踪候选人
- 下一节课会详细讲 ATS 是怎么工作的

---

## 核心洞察

**洞察一：不同场景，简历要打动的人不同**

- 大公司校招：先打动 ATS 和 Recruiter（标准化筛选）
- 中小公司：直接打动 Hiring Manager（他可能亲自看你的简历）
- 社招：展示你能解决 Hiring Manager 的具体问题

**洞察二：应届生投中小公司 = 走社招流程**

- 你在和有经验的人竞争
- 简历要突出项目和潜力，弥补经验不足

**洞察三：简历的目的是拿到面试机会**

- 不是要证明你很厉害
- 而是让看的人觉得：「值得聊一聊」

## 导师寄语

**为什么这节课很重要**

我见过太多求职者把写简历当成一个纯机械的任务——填模板、列职责、然后祈祷有人看上。但简历是沟通。有效的沟通，首先要知道你在跟谁说话。

当你理解了初创公司的 Hiring Manager 可能亲自看每一份简历，而大公司的招聘机器会用自动化系统处理成千上万份——你的策略就会变。你选的词会变。你排列 bullet points 的顺序会变。

**几个关键认知**：

- 没有「放之四海皆准的好简历」。只有在特定场景下、打动特定读者的简历。
- 大多数应届生低估了自己在校招体系外投递时，和有经验者竞争的激烈程度。
- 招聘流程不是「发生在你身上的事」，而是一个你可以理解、可以策略性应对的系统。

## 快速参考

**三种招聘场景**：

- 大公司校招 → 标准化筛选 → 打动 ATS + Recruiter
- 中小公司直投 → 和有经验者竞争 → 打动 Hiring Manager
- 社招 → 解决具体问题 → 展示相关成果

**关键角色**：

- Hiring Manager：有需求、做决定的人
- Recruiter：筛简历的守门人
- ATS：自动化筛选系统
