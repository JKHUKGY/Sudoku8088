# ATS and AI-Era Resume Screening

## Objective

Understand how ATS actually works and debunk common myths that waste your job search time. Learn what's really changing in the AI era and how to write resumes for humans, not systems.

Read the tutorial: [Build Your Resume with AI - Lesson 02](https://github.com/easyscale-academy/learn_build_resume-project/tree/02-ATS/)

## Actionable Items

1. Read the full tutorial from start to finish
2. Understand what ATS actually does (tracking, not judging)
3. Learn the five common myths and why they're wrong
4. Understand the real changes in the AI era (volume explosion, AI-assisted screening, behavior signals, resume homogenization)
5. Internalize the core principle: write for humans, not for systems

**Estimated time:** 20-25 minutes

## Checklist

- [ ] **Understand ATS basics** - Can explain that ATS is a database for tracking, not an AI that rejects candidates
- [ ] **Debunk the myths** - Can identify why keyword stuffing, white font tricks, and "ATS scores" are counterproductive
- [ ] **Know the AI-era changes** - Can explain application volume explosion, AI-assisted screening, and resume homogenization
- [ ] **Adopt the core principle** - Understand why clarity and human readability matter more than keyword optimization

## Submission & Verification

This is a reading-only lesson with no code or deliverables. When you feel confident about the concepts, proceed to the next lesson.

## Grading Rubric

- **Comprehension check:** Ask the student to explain why "beating the ATS" is a misconception
- **Myth identification:** Student can explain why common ATS optimization advice (scores, white font, keyword stuffing) is wrong
- **AI-era understanding:** Student can articulate what's actually changing with AI in hiring (volume, not automatic rejection)
- **Practical application:** Student can describe what "write for humans" means in practice
